<reglas-criticas-tecnicas>
Estas reglas existen para evitar bugs reales y entregar un archivo que funcione a la

1. CHART.JS INCRUSTADO DENTRO DEL ARCHIVO (obligatorio, no CDN):

     a) En el sandbox de código:
        `cd /tmp && mkdir -p cjs && cd cjs && npm install chart.js@4.4.1 --no-audit --no-fund`
     b) Lee el fichero `/tmp/cjs/node_modules/chart.js/dist/chart.umd.js` (≈205 KB,
        build UMD standalone que ya bundliza @kurkle/color).
     c) Antes de incrustarlo en el HTML, sanea cualquier aparición interna de
        `</script` reemplazándola por `<\/script` para evitar cierres prematuros.
     d) En el HTML final, NO uses ningún `<script src="…chart.js…">`. Coloca el
        contenido de chart.umd.js dentro de un `<script>…aquí dentro…</script>` propio,
        en `<head>`, antes de tu propio script de inicialización.

2. ESCRITURA ROBUSTA DEL HTML (obligatorio):
   - Cuando escribas el HTML desde Python en bash, usa SIEMPRE
     `with open(path, "w", encoding="utf-8") as f: f.write(html)`.
   - NO uses `open(path,"w").write(html)` sin context manager: en archivos grandes
     puede no flushear y el archivo queda truncado (sin los últimos KB), lo que rompe
     el cierre del `<script>` y deja la página inservible.

3. VERIFICACIÓN FINAL OBLIGATORIA antes de entregar (en bash):
   - Tamaño del archivo > 290 KB (con Chart.js incrustado).
   - `<script` count == `</script` count (debe haber exactamente 2 de cada: uno para

   - El archivo termina exactamente con `</html>` + salto de línea.
   - Extrae los dos bloques `<script>` por separado y ejecuta `node --check` sobre

   - Conteo de `<canvas>` == número de gráficos que has declarado (mínimo 5).
   - Conteo de elementos `class="finding"` > 30.
   - `<style>`/`</style>`, `<body>`/`</body>`, `</html>` presentes una sola vez.
   Si alguna verificación falla, corrige el archivo antes de entregar.
